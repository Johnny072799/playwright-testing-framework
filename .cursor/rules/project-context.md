# Project Context

This E2E framework is built for **contracting and handoff**.

**Primary use case:** The author implements this framework at companies that do not yet have automation, then trains their dev teams and hands it off for them to maintain.

**Implications for all work on this repo:**
- Prioritize **maintainability** and **clarity** over cleverness or brevity
- Write code and docs with **handoff in mind**—assume the maintainer is new to the framework
- Favor **explicit, training-friendly** explanations in README, comments, and structure
- Keep patterns **consistent** so new devs can learn once and apply everywhere
- When adding features, consider: *Can a newly trained dev understand and extend this without the original author?*
